# 주식 매매 에이전트 (Stock Trading Agent)

LangGraph 기반의 국내 주식 자동 분석 및 매매 추천 시스템입니다.
KOSPI 주요 20개 종목의 실시간 데이터를 수집하고, 기술적 분석을 통해 매수/매도 신호를 생성합니다.

## LangGraph 파이프라인 플로우

에이전트는 5단계의 순차적 파이프라인으로 구성됩니다.
각 단계는 LangGraph의 노드(Node)로 정의되며, 상태(State)를 통해 데이터를 전달합니다.

```
┌─────────────────────────────────────────────────────────┐
│                      START                              │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  1단계: data_collection (데이터 수집)                      │
│  ─────────────────────────────────────────────────────  │
│  • FinanceDataReader로 KOSPI 주요 20종목 OHLCV 수집       │
│  • 최근 60일(거래일 기준) 데이터 가져오기                    │
│  • 출력: stock_data (종목별 DataFrame 딕셔너리)            │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  2단계: technical_analysis (기술적 분석)                    │
│  ─────────────────────────────────────────────────────  │
│  • SMA(5일/20일), RSI(14일), MACD, 볼린저밴드 계산         │
│  • 지표 기반 점수 산정 (-6 ~ +6)                          │
│  • 매매 신호 판정: STRONG_BUY / BUY / HOLD / SELL / ...  │
│  • 출력: analysis_results (종목별 분석 결과 리스트)          │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  3단계: chart_generation (차트 생성)                       │
│  ─────────────────────────────────────────────────────  │
│  • 상위 10개 종목의 캔들차트 생성                           │
│  • 이동평균선, 볼린저밴드, RSI, MACD 서브차트 포함           │
│  • PNG 파일로 output_charts/ 디렉토리에 저장               │
│  • 출력: 차트 파일 경로                                   │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  4단계: recommendation (매매 추천)                         │
│  ─────────────────────────────────────────────────────  │
│  • 분석 결과를 매수/매도/관망 그룹으로 분류                   │
│  • 상위 3개 매수 추천 종목(top_picks) 선정                  │
│  • 추천 근거(reasons) 목록 첨부                            │
│  • 출력: recommendations (추천 결과 딕셔너리)               │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  5단계: trading_decision (매매 결정)                       │
│  ─────────────────────────────────────────────────────  │
│  • 초기 투자금(1,000만원) 기반 매수 수량 계산                │
│  • 종목당 최대 20% 비중 제한                               │
│  • 최대 보유 종목 수(5개) 제한                              │
│  • 보유 종목 중 매도 신호 발생 시 매도 결정                   │
│  • 출력: trade_decisions (매매 주문 리스트)                 │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│                       END                               │
│  → print_report()로 최종 리포트 출력                       │
└─────────────────────────────────────────────────────────┘
```

### LangGraph 상태 (TradingState)

각 노드 간 전달되는 공유 상태 구조:

| 필드 | 타입 | 설명 |
|------|------|------|
| `messages` | `list` | LangGraph 메시지 히스토리 |
| `stock_data` | `dict` | 종목코드 → OHLCV DataFrame 딕셔너리 |
| `analysis_results` | `list` | 종목별 기술적 분석 결과 리스트 |
| `recommendations` | `dict` | 매수/매도/관망 분류 및 top_picks |
| `portfolio` | `dict` | 현금, 보유종목, 총평가금액 |
| `trade_decisions` | `list` | 최종 매매 주문 (종목, 수량, 가격) |
| `phase` | `str` | 현재 처리 단계 |

---

## 각 노드 상세 설명

### 1. data_collection_node

- **파일**: `agents/nodes.py`
- **역할**: FinanceDataReader 라이브러리를 통해 KRX 시장에서 OHLCV(시가/고가/저가/종가/거래량) 데이터를 수집
- **대상**: `config/settings.py`의 `STOCK_UNIVERSE`에 정의된 20개 종목
- **기간**: 최근 60일 (설정 변경 가능)
- **에러 처리**: 개별 종목 수집 실패 시 경고 출력 후 스킵

### 2. technical_analysis_node

- **파일**: `agents/nodes.py`, `strategies/technical.py`
- **역할**: 수집된 OHLCV 데이터에 5가지 기술적 지표를 계산하고, 점수를 합산하여 매매 신호 생성
- **신호 판정 기준**:
  - `score >= 3` → **STRONG_BUY**
  - `score >= 1` → **BUY**
  - `score == 0` → **HOLD**
  - `score <= -1` → **SELL**
  - `score <= -3` → **STRONG_SELL**

### 3. chart_generation_node

- **파일**: `agents/nodes.py`, `charts/plotter.py`
- **역할**: mplfinance 라이브러리로 캔들차트 + 기술적 지표 차트를 생성
- **차트 구성** (4개 패널):
  1. **메인 패널**: 캔들차트 + SMA5(주황) + SMA20(파랑) + 볼린저밴드(회색 점선)
  2. **거래량 패널**: 봉 차트 (매수: 초록, 매도: 빨강)
  3. **RSI 패널**: RSI 14일 (보라색)
  4. **MACD 패널**: MACD(파랑) + Signal(빨강) + Histogram(막대)
- **저장**: `output_charts/{종목코드}_{종목명}.png` (150 DPI)

### 4. recommendation_node

- **파일**: `agents/nodes.py`
- **역할**: 분석 점수를 기반으로 매수/매도/관망을 분류하고, 상위 매수 추천 종목 3개를 선정
- **출력 항목**: 종목명, 현재가, 등락률, 신호, 점수, 추천 근거

### 5. trading_decision_node

- **파일**: `agents/nodes.py`
- **역할**: 포트폴리오 규칙을 적용하여 실제 매매 주문을 생성
- **적용 규칙**:
  - 종목당 최대 포트폴리오의 20%까지 투자
  - 최대 5개 종목 동시 보유
  - 손절 -5% / 익절 +10% 기준
  - 매도 신호 발생 보유종목은 전량 매도

## 기술적 분석 지표

### 점수 산정 기준표

| 지표 | 조건 | 점수 | 설명 |
|------|------|------|------|
| **SMA** | 골든크로스 발생 (SMA5가 SMA20 상향 돌파) | +2 | 강한 매수 |
| | 데드크로스 발생 (SMA5가 SMA20 하향 돌파) | -2 | 강한 매도 |
| | SMA5 > SMA20 (상승 추세 유지) | +1 | 약한 매수 |
| | SMA5 < SMA20 (하락 추세 유지) | -1 | 약한 매도 |
| **RSI** | RSI < 30 (과매도) | +2 | 반등 기대 |
| | RSI > 70 (과매수) | -2 | 조정 기대 |
| **MACD** | MACD가 Signal을 상향 돌파 | +2 | 매수 전환 |
| | MACD가 Signal을 하향 돌파 | -2 | 매도 전환 |
| | MACD Histogram > 0 | +1 | 상승 모멘텀 |
| | MACD Histogram < 0 | -1 | 하락 모멘텀 |
| **볼린저밴드** | 종가 < 하단밴드 | +2 | 과매도 반등 |
| | 종가 > 상단밴드 | -1 | 과열 주의 |
| **거래량** | 거래량 > 평균의 2배 & 매수 신호 | +1 | 거래량 동반 상승 |

## 출력 결과 해석

### 매매 신호

| 신호 | 의미 | 권장 행동 |
|------|------|-----------|
| **STRONG_BUY** | 여러 지표가 강한 매수 신호 (점수 3+) | 적극 매수 고려 |
| **BUY** | 매수 신호 우세 (점수 1~2) | 매수 고려 |
| **HOLD** | 방향성 불분명 (점수 0) | 관망 |
| **SELL** | 매도 신호 우세 (점수 -1~-2) | 매도 고려 |
| **STRONG_SELL** | 여러 지표가 강한 매도 신호 (점수 -3 이하) | 적극 매도 고려 |

### 차트 읽는 법

- **초록 캔들**: 양봉 (종가 > 시가, 상승)
- **빨간 캔들**: 음봉 (종가 < 시가, 하락)
- **주황 선 (SMA5)**: 5일 이동평균 — 단기 추세
- **파란 선 (SMA20)**: 20일 이동평균 — 중기 추세
- **회색 점선**: 볼린저밴드 상단/하단
- **RSI 50 이상**: 상승 모멘텀 / 50 이하: 하락 모멘텀
- **MACD 파란선이 빨간선 위**: 매수 구간 / 아래: 매도 구간

## 주의사항

> **본 프로그램은 기술적 지표 기반의 참고용 분석 도구입니다.**
> 투자 판단은 반드시 본인의 책임하에 이루어져야 하며,
> 이 프로그램의 추천 결과로 인한 투자 손실에 대해 어떠한 책임도 지지 않습니다.

- 실시간 매매가 아닌 **일봉 기준 분석**입니다.
- KRX 데이터 서버 상태에 따라 데이터 수집이 실패할 수 있습니다.
- 기술적 분석만으로는 기업의 펀더멘털, 시장 뉴스, 거시경제 등을 반영하지 못합니다.
- 실제 투자 전 반드시 추가적인 분석을 병행하시기 바랍니다.
